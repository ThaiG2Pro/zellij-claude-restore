## 4. Regression Focus — BR-tests-008 Four Named Areas

### (a) Snap-pane neutralization — `start_suspended`/`close_on_exit` child node stripping

Code: `neutralize_snap_pane()` at `enrich.rs:85–127` strips `command`/`start_suspended`/`close_on_exit`
from both `entries_mut()` (property form) and `children_mut()` (child-node form). The spec identifies
these as CHILD nodes in real Zellij dumps; the code correctly handles both forms.

Tests: `snap_pane_timeout_all_children_stripped_ac_tests_010` (integration, 4 negation asserts) and
`neutralize_snap_pane_strips_all_child_nodes_ac_tests_010` (unit, child-node loop). Both PASS.

Verdict: COVERED. The Jun 29 regression (start_suspended on command-less pane errors on restore) is
protected by two layers.

### (b) Idempotent enrichment — no double-inject

Code: `pane_has_session_id()` at `enrich.rs:168–182` checks for `--resume` OR `--session-id` in the
`args` child node and returns early in `maybe_enrich_pane` if found.

Tests: `existing_resume_not_double_injected_ac_tests_013` uses count assertion (`assert_eq!(count, 1)`).
`idempotent_double_enrichment_ac_tests_015` uses `assert_eq!(first, second)` — the strongest possible
test for this scenario. Both PASS.

Verdict: COVERED strongly.

### (c) Template-subtree skip

Code: `is_template_node()` at `enrich.rs:34–39`; `entering_template` flag propagated in `enrich_nodes`.
All four template names (`new_tab_template`, `tab_template`, `swap_tiled_layout`, `swap_floating_layout`)
covered.

Tests: `all_template_node_types_skipped_ac_tests_016_extended` iterates all 4 template types and asserts
no `--resume` injected. Contrast test `claude_pane_outside_template_is_enriched_ac_tests_017` confirms
the skip is not a global no-op. Both PASS.

Verdict: COVERED.

### (d) Relative/absolute cwd resolution

Code: `resolve_cwd()` at `enrich.rs:160–166`. Four branches: absolute passthrough, relative join with
trailing-slash trim, no-pane-cwd inherits base, both-None returns None.

Tests: 4 unit tests with `assert_eq!` exact equality cover all branches. Integration test

## 8. Deviation Review

### DEV-001: `#[cfg(not(test))]` guards in `main.rs`

**Assessment: Behavior-neutral. Confirmed.**

The guards at `main.rs:3–6` (imports) and `main.rs:15–19` (struct/impl), and `main.rs:65–66`
(register_plugin!) are transparent to the `wasm32-wasip1` target because `cfg(test)` is NEVER set for
WASM targets. The WASM build produces the identical binary as without these guards. The guards only
affect the native host compile path used by `cargo test`.

The `#[cfg_attr(test, allow(dead_code))]` on `resolve_session_uuid` is also behavior-neutral: the
function is compiled and linked in WASM builds; the attribute only suppresses a dead-code lint warning
on the native test target where the function is unreachable (the `State`/`pipe()` code calling it is
cfg-gated out).

This is idiomatic Rust for binary crates with platform-specific host dependencies. No design change
required; ADR-002's statement ("inline tests in a binary crate run via `cargo test` without any
crate-type change") remains correct — the crate type was not changed.

**Verdict: Behavior-neutral. Not a bug.**

### IMPL-003: `enrich_nodes` signature `&mut [KdlNode]` (was `&mut Vec<KdlNode>`)

**Assessment: Correct and behavior-neutral. Confirmed.**

`enrich_nodes` only iterates the nodes; it never calls `push`/`pop`/`remove` on the slice itself (child
mutation happens via `node.children_mut()` on individual nodes). The `&mut [KdlNode]` → `&mut Vec<KdlNode>`
coercion via `Deref` at all call sites (`doc.nodes_mut()`, recursive `children.nodes_mut()`) is
transparent. This satisfies `clippy::ptr_arg` which is a correctness lint (it fires when `&mut Vec` is
passed but only slice operations are used, indicating a needlessly restrictive signature).

**Verdict: Correct. Not a bug.**
