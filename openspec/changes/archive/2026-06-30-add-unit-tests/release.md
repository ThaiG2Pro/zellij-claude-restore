## Release Summary

**Type**: Refactor-for-testability with byte-identical runtime behavior. No user-visible behavior change.

This release extracts the pure KDL-enrichment logic from `src/main.rs` into a standalone
`src/enrich.rs` module, adds a 33-test regression suite covering all historically-broken behaviors,
and gates every push/PR with a new CI workflow. The `.wasm` output and the enrichment behavior are
parity-identical to the pre-extraction code (AC-tests-007, human-confirmed 2026-06-30).

## What Changed

### 1. New file: `src/enrich.rs` — pure KDL-enrichment module (AC-tests-001/002/003)

The eight KDL-enrichment functions are extracted from `src/main.rs` into a new `src/enrich.rs`
module with zero `zellij_tile` and zero `std::fs` imports. A `SessionResolver` closure seam
(`&dyn Fn(&str) -> Option<String>`) is threaded through `enrich_claude_panes` → `enrich_nodes` →
`maybe_enrich_pane` so tests can stub the marker lookup without filesystem access (ADR-001).

Moved functions: `enrich_claude_panes` (pub), `enrich_nodes`, `neutralize_snap_pane`, `resolve_cwd`,
`pane_has_session_id`, `inject_session_id`, `basename`, `is_template_node` (all pub(crate)).

Key invariants preserved (AC-tests-004/005/006):
- Binary crate (`[[bin]]`); `register_plugin!` still emits `_start`. Never cdylib.
- KDL v1 only: `parse_v1` / `ensure_v1` — the default v2 `parse()` is never used.
- Injection uses `--resume`; `--session-id` is detection-only (already-pinned guard).

Stale doc-comment corrected: `--session-id` → `--resume` in `enrich_claude_panes` doc.

### 2. Modified: `src/main.rs` — re-wired to `mod enrich;`

`mod enrich;` added at line 1. The eight functions replaced by `enrich::enrich_claude_panes(...)`.
`#[cfg(not(test))]` guards added around `use zellij_tile`, `struct State`, `impl ZellijPlugin`,
and `register_plugin!` — required to isolate WASI host symbols from the native test linker (DEV-001,
behavior-neutral on the `wasm32-wasip1` target). Synchronous dump→enrich→save flow unchanged (ADR-004,
BR-tests-006).

### 3. New file: `.github/workflows/ci.yml` — push/PR CI (AC-tests-025/026/027/028)

Separate from `release.yml` (which stays tag-only, unchanged). Triggers on `push` and `pull_request`.
Single `ubuntu-latest` job; sequential fail-fast steps:

1. `rustup update stable` + `rustup target add wasm32-wasip1`
2. `cargo build --release --target wasm32-wasip1`
3. `cargo fmt --check`
4. `cargo clippy -- -D warnings`
5. `cargo test`

No coverage gate (R-COV-001 scoped out per `context/stack.md`, ADR-003/D-tests-005).

### 4. Regression test suite — 33 tests (AC-tests-007 through 028)

`#[cfg(test)] mod tests` inline in `src/enrich.rs`. All test names include their AC-ID.
Coverage of four historically-broken regression areas (BR-tests-008):

| Area | Tests | Assertion strength |
|------|-------|--------------------|
| Snap-pane neutralization (`start_suspended`/`close_on_exit` child-node stripping) | `snap_pane_timeout_all_children_stripped_ac_tests_010`, `neutralize_snap_pane_strips_all_child_nodes_ac_tests_010` | 4 negation asserts + child-node loop |
| Idempotent enrichment (no double-inject) | `existing_resume_not_double_injected_ac_tests_013` (count==1), `idempotent_double_enrichment_ac_tests_015` (assert_eq! first==second) | Strongest possible — count + equality |
| Template-subtree skip (all 4 template types) | `claude_pane_in_template_not_enriched_ac_tests_016`, `all_template_node_types_skipped_ac_tests_016_extended`, `claude_pane_outside_template_is_enriched_ac_tests_017` | All 4 types; contrast test confirms skip is not a global no-op |
| Relative/absolute cwd resolution | 4 unit tests (exact `assert_eq!`) + `relative_cwd_resolved_and_enriched_br_tests_008` (RefCell captures resolver argument) | Exact equality on all branches |

Parity test: `parity_claude_pane_enriched_with_resume_ac_tests_007` uses
`assert_eq!(result, PARITY_EXPECTED)` against a `const` byte-identical expected string (F-001
closed in S4-fix; PARITY_EXPECTED contains `\/`-escaped cwd and the injected `args "--resume"
"00000000-...";` child node).

## Post-Deploy Smoke Test

### Step 1 — Build and deploy the WASM artifact

```bash
cargo build --release --target wasm32-wasip1
cp target/wasm32-wasip1/release/zellij-claude-sync.wasm ~/.config/zellij/plugins/
```

### Step 2 — Purge the Zellij plugin cache (REQUIRED)

Zellij caches the compiled plugin per session and does NOT invalidate on file-content change.
Skipping this step means the old plugin keeps running and changes appear to have no effect.

```bash
find ~/.cache/zellij -type d -name 'zellij-claude-sync.wasm' -prune -exec rm -rf {} +
```

Verify the deployed file is the new build:

```bash
md5sum target/wasm32-wasip1/release/zellij-claude-sync.wasm \
       ~/.config/zellij/plugins/zellij-claude-sync.wasm
# Both hashes must match.
```

### Step 3 — Test in a FRESHLY STARTED Zellij session

Do NOT test in a reattached session — it keeps the old compiled plugin in memory.

```bash
zellij -s smoke-test-$(date +%s)
```

Inside that session, run `snap <name>` on a layout with at least one `claude` pane. The first
invocation recompiles the wasm (a few seconds; `timeout 3` in `snap` may expire — just run `snap`
again). Confirm:

1. `~/.config/zellij/layouts/<name>.kdl` is written.
2. The KDL contains `args "--resume" "<uuid>"` for the claude pane (not `--session-id`).
3. `zellij --layout <name>` restores the layout and the claude pane opens the correct conversation.

### Step 4 — Confirm CI is green

The new `ci.yml` runs on the PR itself. Before merging, confirm all 5 CI steps pass (green
checkmark on the `feature/add-unit-tests` → `main` PR). This is the primary gate for AC-tests-025–028.
